Jenkins+Git+Gitlab+Ansible 实现持续集成自动化部署

为了减少开发人员进行代码提交时出现错误，消耗大量沟通成本和时间成本，导致公司进度拖延，经过讨论 决定部署自动集成环境来实现产品上线流程化，加快公司进度。具体流程为：开发人员编写代码，将代码上传到 GitLab 服务器，Jenkins 从 GitLab 服务器上面拉取最新代码数据到本地，根据 Jenkins 项目配置，将代码拉取 到本地后，可以进行编译、测试、打包等工作，最后将数据包写入到共享服务器，应用服务器最后从共享服务器 拉取数据实现上线功能。

项目职责： 在此次项目中我负责并参与了： 1、Jenkins、GitLab 和 Ansible 等服务器的搭建部署工作； 2、JenKins 相关插件的安装与 GitLab 的账号配置关联； 3、实现了 JenKins 自动从 GitLab 仓库拉取代码并打包上传到共享服务器； 4、实现 Ansible 服务器从共享服务器自动下载并解压代码部署到应用服务器，实现了 CI/CD。

项目名称：VMware 虚拟化项目

项目描述： 由于公司业务的不断扩张，但机房的空间不足，同时现有的部分服务器存在资源（CPU、内存、硬盘等）利 用率低以及部署回收资源效率差、不能统一管理等问题，所以采用 VMware 的虚拟化技术，既可以解决机房空间 不足的问题，同时可以给项目组快速灵活的部署相应的虚拟机，大大提高了服务器资源的利用率。

项目职责： 在此次项目中我前期参与部署了 4 台 ESXI 6.5 和 1 台 Vcenter，Vcenter 统一接管了四台 ESXI 6.5 主机，然 后挂载了存储，可以通过管理平台创建多业务的系统模板，可以根据业务需求快速部署相应配置的虚拟机。四台 ESXI 主机实现了高可用性，即其中一台出现问题，它上面的虚拟机能够快速自动迁移到其他正常的主机上，不会 使业务受到影响。将一些老旧的服务器业务安全无损的迁移到了 Vcenter 虚拟机上。

项目名称：kubernetes +Ceph 集群搭建与维护

项目描述： 随着技术的不断升级发展，以前的传统模式的开发部署可能无法满足现在的快速化节奏，所以决定将部分业 务由传统的服务器主机部署逐渐迁移到 docker 容器，存储则使用分布式存储 Ceph。容器所具有的优点是显而 易见的，不仅所需要资源少，而且高效迅速。开发人员无需再配置复杂的运行环境，只需做好自己的本职工作， 可以做到一次构建，到处运行。Kubernetes 用来管理这些容器，再配合 Ceph 这种稳定高效的分布式存储，极 大的提高了产品应用的更新速度。

项目职责： 参与构建 k8s+Ceph 集群搭建方案， 根据方案总共使用八台服务器，其中四台性能较好的用来搭建部署了 k8s 集群，一台 Master 主机，三台 node 节点；另外的四台硬盘容量比较大，用来搭建 Ceph 集群，将 Ceph 跟 K8S 进行集成，最后配置所需要的监控。

项目名称：Zabbix+Grafana 监控服务器和网络设备

项目描述： 由于机房服务器以及各部门交换机、终端台式机等网络设备的不断增加，现有的监控平台存在设备授权数量 限制以及不能灵活的自定义监控内容和报警方式等问题，所以采用了开源的 Zabbix 系统监视平台。zabbix server 可以通过 SNMP，zabbix agent，ping，端口监视等方法提供对远程服务器/网络状态的监视，数据收集等功能， 它可以运行在 Linux Server，Windows Server 等平台上，配合 Grafana 的人性化的图形化界面分析展示，从而 能够很好的满足公司业务需求。

项目职责： 在此次项目中我搭建了 Zabbix 和 Grafana Server，优化配置了 Zabbix 自动发现规则以及监控模板，从而 能够监测公司的服务器、交换机等网络设备的工作状态。另外对于一些重要的业务如 Tomcat、MySql、Oracle、 K8s、docker 等也配置了相关的模板，通过客户端或者 snmp 的方式收集分析这些服务的信息，然后通过 Grafana 将收集到的数据进行可视化界面展示到外面的监控大屏。
